## 输出要求

### 🚨 1. 信息真实性要求（最重要！）
- **必须**使用项目信息中的真实数据（招标人、联系人、电话等）
- **必须**使用企业资料中的真实信息（公司名称、资质、业绩等）
- **严禁编造**任何人名、公司名、电话号码、证书编号等具体信息
- 信息不足时，使用【待补充：具体字段名】标记，**绝不编造**

### 2. 格式要求
- 输出HTML格式的章节内容
- 使用标准HTML标签：`<p>`、`<ul>`、`<li>`、`<h3>`、`<table>`、`<tr>`、`<th>`、`<td>`等
- 不要输出章节标题，只输出正文内容
- **如果用户明确要求表格格式，必须使用HTML表格标签**（`<table>`、`<thead>`、`<tbody>`、`<tr>`、`<th>`、`<td>`）

### 3. 内容要求
- 内容至少{{min_words}}字，分为3-6段
- 确保内容完整、逻辑清晰、论述充分

### 4. 章节类型指导
根据标题类型生成合适内容：

- **格式类章节**（投标函、授权书等）
  → 生成对应的格式范本，包含必要的格式要素
  → **必须**使用真实的项目信息（招标人、项目名称等）和企业信息（公司名称、法定代表人等）
  → **严禁**使用"XX公司"、"XXX招标人"等占位符

- **技术方案类章节**
  → 详细描述技术路线、实施方法、保障措施
  → 体现技术可行性和先进性
  → 可以基于行业标准生成通用方案，但涉及企业具体信息时必须使用真实资料

- **商务类章节**
  → 说明报价依据、优惠措施、付款方式
  → 展现商务优势和合理性
  → **项目相关信息（招标人、项目名称）必须使用真实数据**

- **公司/业绩类章节**
  → **必须**充分利用企业资料展示实力，**严禁编造**
  → 突出相关项目经验和成功案例（从企业资料中提取）
  → 如果企业资料不足，标注【待补充：公司介绍/业绩案例】，不得编造
