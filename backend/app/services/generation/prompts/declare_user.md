## 输出要求

### 1. 格式要求
- 输出完整的申报书章节内容（**HTML格式**）
- 不要输出章节标题，只输出正文内容
- 使用标准HTML标签：`<p>`、`<ul>`、`<li>`、`<h3>`、`<h4>`、`<strong>`、`<table>` 等
- **不要使用Markdown格式**，必须是规范的HTML

### 2. 内容要求
- 内容必须完整、专业、符合评审标准
- 结构清晰，逻辑严密，语言规范
- 紧扣申报要求，突出项目优势
- 每段使用 `<p>` 标签包裹
- 列表使用 `<ul>` 和 `<li>` 标签
- 重点内容使用 `<strong>` 标签加粗

### 3. HTML格式示例
```html
<p>这是第一段内容，介绍企业基本情况。</p>

<h3>主要优势</h3>
<ul>
  <li><strong>技术优势</strong>：具备国内领先的技术实力</li>
  <li><strong>人才优势</strong>：拥有专业团队50余人</li>
  <li><strong>项目经验</strong>：成功实施多个重大项目</li>
</ul>

<p>企业在行业内具有显著竞争力，具备承担本项目的充分能力。</p>
```

### 5. 质量标准
在内容末尾添加置信度注释：
- **HIGH（高）**：基于详细的用户资料生成，内容真实可靠
- **MEDIUM（中）**：部分基于用户资料，部分基于行业标准扩展
- **LOW（低）**：主要基于行业标准和最佳实践生成

**注释格式**：`<!-- 置信度：HIGH -->`
