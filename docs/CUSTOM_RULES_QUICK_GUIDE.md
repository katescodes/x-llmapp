# 自定义规则管理 - 快速使用指南

## 入口位置

在招投标工作台左侧，"格式模板"按钮下方，有一个**"自定义规则"**按钮（粉色渐变）。

## 创建自定义规则

### 步骤1：进入规则管理页面
点击左侧的"自定义规则"按钮

### 步骤2：创建规则包
1. 点击右上角的"+ 创建规则包"按钮
2. 输入规则包名称，例如："特殊资格要求"
3. 输入规则要求（自然语言），例如：
   ```
   1. 投标人必须具有有效的营业执照，且注册资本不低于500万元
   2. 投标人必须提供近三年的财务审计报告
   3. 投标报价不得高于预算的110%
   4. 投标文件必须按照规定格式装订，页码连续
   ```
4. 点击"创建规则包"按钮
5. 系统会自动使用AI分析规则要求，生成结构化规则

### 步骤3：查看规则
1. 在左侧规则包列表中选择刚创建的规则包
2. 右侧会显示该规则包的所有规则详情
3. 每条规则包含：
   - 规则名称和ID
   - 维度（资格审查/技术规格/商务条款等）
   - 严重程度（低/中/高）
   - 执行器类型（确定性/LLM语义）
   - 是否硬性要求（废标项）
   - 条件配置（JSON格式）

## 应用规则到审核

### 步骤1：进入审核页面
点击"⑤ 审核" Tab

### 步骤2：选择规则包
1. 在"可选：选择自定义规则包"部分
2. 勾选要应用的一个或多个规则包
3. 每个规则包旁边会显示规则数量

### 步骤3：运行审核
1. 选择投标人
2. （可选）同时选择自定义规则文件资产
3. 点击"开始审核"按钮
4. 系统会应用选中的规则包进行审核

## 管理规则包

### 查看规则包列表
- 左侧显示所有规则包
- 显示规则包名称和规则数量
- 点击可查看详情

### 删除规则包
- 点击规则包右侧的"删除"按钮
- 确认删除
- 规则包和其中的所有规则都会被删除

## 规则示例

### 输入示例
```
投标人必须具有有效的营业执照，且注册资本不低于500万元
```

### AI生成的规则（示例）
1. **营业执照检查**
   - 维度：资格审查
   - 执行器：LLM语义
   - 严重程度：高
   - 硬性要求：是

2. **注册资本检查**
   - 维度：资格审查
   - 执行器：确定性
   - 严重程度：高
   - 硬性要求：是

## 注意事项

1. **规则分析依赖AI**
   - 需要配置可用的LLM模型
   - AI分析可能需要几秒钟时间
   - 如果分析失败，规则包会创建但不包含规则

2. **规则是项目级的**
   - 每个项目的规则包是独立的
   - 切换项目会看到不同的规则包列表

3. **规则包可组合**
   - 可以创建多个规则包，针对不同场景
   - 在审核时可以灵活选择一个或多个规则包

4. **规则不可编辑**
   - 当前版本不支持编辑已生成的规则
   - 如需修改，请删除规则包后重新创建

## 常见问题

**Q: 规则包创建后，在哪里应用？**
A: 在"⑤ 审核" Tab中，选择要应用的规则包，然后开始审核。

**Q: 可以同时应用多个规则包吗？**
A: 可以，在审核页面可以勾选多个规则包。

**Q: 规则包和规则文件资产有什么区别？**
A: 
- 规则包：AI分析生成的结构化规则，可精确执行
- 规则文件资产：上传的文档，作为额外上下文参考

**Q: AI生成的规则不准确怎么办？**
A: 当前版本暂不支持编辑，建议删除后重新创建，并优化规则要求的描述。

**Q: 规则包可以在不同项目间共享吗？**
A: 当前版本不支持，规则包是项目级的。

## 技巧和建议

### 编写好的规则要求
1. **清晰明确**：每条规则描述一个具体要求
2. **包含数值**：如果涉及数值比较，明确写出数值和单位
3. **分类编号**：使用编号区分不同的规则
4. **完整表述**：包含必要的上下文信息

### 规则包的组织
1. **按场景分类**：例如"资格要求"、"技术规格"、"商务条款"
2. **粒度适中**：每个规则包5-10条规则为宜
3. **命名规范**：使用清晰的名称，便于识别

### 审核时的选择
1. **基础规则**：创建一个包含通用要求的规则包
2. **特殊规则**：针对特定项目创建额外的规则包
3. **灵活组合**：根据实际情况选择应用哪些规则包
